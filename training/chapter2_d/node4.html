File System Security

On most Linux systems users have a private directory which is usually "/HOMEDIRECTORYPATH/username". A path is the series of directories that indicate the location of a file. For our purposes we will assume that "HOMEDIRECTORYPATH" is just the directory
/home/username. (On some systems this may be more complicated, for instance the default on Solaris is
/exports/home/username.)Linux allows users to set the security options of every file. When we do a listing of /home/frodo, we get the following output. 

[frodo@macros frodo]$ ls -lR
.:
total 12
drwxrwxr-x    2 frodo    users  ... bcCode
-rw-r--r--    1 frodo    users  ... bcCode.tar.gz
                                                                                
./bcCode:
total 16
-rw-r--r--    1 frodo    users  ... hello
-rw-rw-r--    1 frodo    users  ... hello.c
[frodo@macros frodo]$
A few of the fields of this listing have been removed in the interest of conserving space. Let's take a look at some of the parts of this listing. The term
"[frodo@macros frodo]$"
is the prompt. This string is formated in the shell's configuration files.(see Section ) Here, "frodo" refers to the account name, and "macros" is the name of the server. The string
"ls -lR"
is a call to the command "ls", which is the command for listing files and/or directories. The "-lR" are the options. This command/option structure is standard for Linux. We will see more about this when we discuss the "man" or help command. In this case the "l" in "-lR" means give the "long" or detailed listing, and the "R" makes the listing recursive so that it will list files in the subdirectories. We will be exploring other options to the ls command in Exercises 2.2.3. The first group of files are those directly in Frodo's home directory. You will notice that the line for bcCode starts with a d. This indicates that this file is a directory. After the d you see the characters rwxrwxr-x these characters are the permissions field. The permissions come in three groups of three letters. The groups are owner, group, and world. The possible letters are r, w and x which stand for read permissions, write permissions, and execute permissions respectively. According to this output the owner Frodo and the group users, can read, write and execute this directory.So you may be wondering why a directory needs execution permission. The answer is that giving a user or group execute permissions on a directory means that they can cd or move into that directory. The last three characters on this line r-x means that the world (another way of saying all users) have read and execute permissions on this directory, but they cannot write to it. This means that any user can cd to this directory and list the files, but they cannot create or delete files.

The file permissions are changed with the chmod command. You can use the letters u,g,o,a for users, group, others, or all and the letters w,r,x for read write and execute. A + adds permissions and a - subtracts permissions. For example if Frodo wanted to let all users have write permissions on his directory bcCode (a very bad idea), he would give the command:

chmod o+w bcCode

The corresponding line in the above listing would become:

drwxrwxrwx 2 frodo users ... bcCode
