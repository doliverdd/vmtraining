Exercises

  1. Read the man page for the ls command (used to list files and directories), and do the following.
    1. Do a listing of your home directory.
    2. Do a long listing of your home directory and find out what fields are missing from the example in the text.
    3. In Linux files begining with a "." are hidden files. These do not show up on the default version of ls. From the man page find out how to list these hidden files.
    4. Find out how to list the last time a file was accessed.
  2. Read the man page for the cp, cd, and tar commands, copy the file bcCode.tar.gz from the \tmp directory and untar it (hint check out the x,v,f,and z options for tar).
  3. The new bcCode directory contains the example code for this course. Change directories into the bcCode directory (You did do the last assignment, right?) Try the following commands.
    1. Make a recursive copy of the HELLO directory, called myHello. You may need to reread the man page for cp. You will use this new directory to make your modifications.
    2. The find command will recursively search directories for a file name. Read the man page for find, and try to find the file hello.c.
    3. Linux does not use the file extensions to determine the type of file. Instead there is a command called file. Change to the myHello/C directory. Do the command "file *" to find out the types of files in this directory.
    4. The cat command allows you to list the contents of a text file. Based on the output from the find command in the previous part of this exercise, which file can you "cat"? Try "catting" that file. Now try typing "cat hello". This should cause your terminal window to fill with junk. Use the reset command to clear up this problem.
  4. There are two interactive listing commands on Linux, "less" and "more". These allow you to move back and forth in the listing and do searches. Less is has more features than more, and so in this case "less is more". Change to the DOC subdirectory of the bcCode directory. Type "less chapter2.tex".

    1. Move up and down in the the file using the arrow keys and/or the Enter key.
    2. What happens when you hit the space bar?
    3. Searches are done in a different "mode". Enter :. Now you should be at the bottom of the window. Type "/less is more" to find the code for this question. This is a forward search.
    4. Type :. Now type "?Exercises" to find the beginning of these exercises. This is a backward search.
