Bash Internal Commands

alias
The alias command allows you to create a short hand or easy mneumonic for longer commands. For example, one of the repetative tasks user frodo might do is to issue a ps command and then search for his own processes. He could make this simpler by "piping" the output of the ps command to the grep command (See Section 4.1.3) by typing :

ps auxw | grep frodo.

If frodo had to do this several times it might become tiresome. Issuing the command :

alias psf="ps auxw | grep frodo"

would allow frodo to get the same results by typing:

psf.

source
Source reads and executes commands from a file. This command is particularly useful when you are making changes to your bash configuration files and you want to use them in your current session. For example to test changes in your .bashrc file you would type source .bashrc. (See Section 4.1.2)
bg,fg,z
When a process is running in a bash shell, you can type in z (Control-z) and the process will be suspended. If you type bg after this the job/process will be put in background mode which will free the shell for other tasks. If you type fg after this it will bring the job back into the forground for this shell. There are more options for job control in the bash shell, but these are the commands that are used most often.
c
Abort the current process.
echo
Output the arguments. This is useful in shell scripting, and in checking the value of the environment variables for example echo $PATH will print out the current value of your PATH varialbe.
export
The arguements are marked for automatic export to the environmentof subsequently executed commands. This is used to set environment variables so that any process called by the shell will also have the variables set. (See Section 4.1.2)
exit
The exit command causes the shell to quit.
history
Displays a list of past commands. This command has many options and can be used to rerun commands. In most bash installaitons,commands can be rerun, and you can scroll through this list by using the up and down arrows.
kill
The kill command sends a signal to a process. The most common use is kill -9 pid which will kill the process with process number pid.
printenv
Display the current state of your environment variables.
pwd
List the current directory.
