A Gentle Introduction to VI

To do any work on a system you must be able to manipulate the contents of files. There is an editor on all *nix systems called VI . Although most modern *nix systems have GUI text editors there are two good reasons for using VI. The VI editor has been included with the *nix systems for quite some time, and it is ubiquitous. Although its interface may not be intuitive, VI is a very powerful editor. Many people find that after they become comfortable with VI that all other editors are somewhat slow. There is not a lot of theory to using an editor, so most of this section will be devoted to leading you through exercises working with VI.

There is one aspect of VI that you should be aware of before you start working the examples. VI is mode based. This means that when working with VI you are in one mode or the other, and which mode you're in will determine the effect of what you type. These modes are command and input. The VI editor starts in command mode. In this mode you can move back and forth in the text, and do searches both forward and backward, and enter editor commands. Input mode is just what it implies, this is the mode for entering text.

There are several different flavors of VI available. The one that you will be using is called vim , or improved VI. We are introducing you to this version because it does syntax highlighting. That is, it will color different parts of programming files based on the syntax of that language. (For the rest of this chapter, when we use the capitalized VI, we mean the program. The command that you will run is vi in lowercase.)
