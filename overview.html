Linux Fundamentals

It is not necessary to become a Linux guru to be able to effectively use these systems. In fact, most of your interaction with the system will be through what is called a shell. A shell is nothing more than a program that makes using the system resources easier. Those of you that have used the "Command Tool" in Windows operating systems have used a shell which is referred to as the DOS shell. Although it may seem awkward at first for those of you that have only used a GUI based system like Windows, it will quickly become second nature. Many people, after becoming comfortable with the command line, find a GUI interface somewhat restrictive. This interface to the system resources is called a command line interface.
The default user shell for Linix is called bash. In this part of the training, you will find out how to access the system, use the command line interface and the shell, work with file and directories, and learn some basic commands.

Getting Started

The first thing that you must know is how to access the system. You should have been given an account name and password when you arrived in the lab today. You will need both to access the system since Linux is a true multiuser system. Please keep the account name and password in a safe place. For our examples we will assume that the system has a user with the account name "frodo". Notice that frodo is not capitalized. This is because Linux is case sensitive. Before we log on to the system let's take a look at the filesystem structure.
The filesystem on Linux is composed of directories and files. The directories are synonymous with folders in the Windows operating systems. The main system directories are displayed below. On most Linux systems the contents of these main directories are fairly standard, although the reasons may not be clear. The common contents of some of the more important directories for users are listed here:
/bin Command binaries
/home Users home directories
/lib System libraries
/sbin System binaries
/tmp Scratch Space
/usr/bin Binaries for user commands
/usr/include Header or include files
/usr/lib Libraries
/usr/local Locally installed binaries,libraries,and includes

Basic File System Commands

File System Security

On most Linux systems users have a private directory which is usually "/HOMEDIRECTORYPATH/username". A path is the series of directories that indicate the location of a file. For our purposes we will assume that "HOMEDIRECTORYPATH" is just the directory /home/usernameinitial/username. When we do a listing of/home/f/frodo, we get the following output.

[frodo@macros frodo]$ ls -lR
.:
total 12
drwxrwxr-x 2 frodo users ... bcCode
-rw-r--r-- 1 frodo users ... bcCode.tar.gz

./bcCode:
total 16
-rw-r--r-- 1 frodo users ... hello
-rw-rw-r-- 1 frodo users ... hello.c
[frodo@macros frodo]$

File permissions

A few of the fields of this listing have been removed in the interest of conserving space. Let's take a look at some of the parts of this listing. The term

[frodo@macros frodo]$
is the prompt. This string is formated in the shell's configuration files. Here, "frodo" refers to the account name, and "macros" is the name of the server. The string

ls -lR
is a call to the command "ls", which is the command for listing files and/or directories. The "-lR" are the options. This command/option structure is standard for Linux. We will see more about this when we discuss the "man" or help command. In this case the "l" in "-lR" means give the "long" or detailed listing, and the "R" makes the listing recursive so that it will list files in the subdirectories. We will be exploring other options to the ls command later in the exercises.
The first group of files are those directly in Frodo's home directory. You will notice that the line for bcCode starts with a "d". This indicates that this file is a directory. After the "d" you see the characters "rwxrwxr-x" these characters are the permissions field. The permissions come in three groups of three letters. The groups are owner, group, and world. The possible letters are r, w and x which stand for read permissions, write permissions, and execute permissions respectively. According to this output the owner Frodo and the group users, can read, write and execute this directory. So you may be wondering why a directory needs execution permission. The answer is that giving a user or group execute permissions on a directory means that they can "cd" or move into that directory. The last three characters on this line r-x means that the world (another way of saying all users) have read and execute permissions on this directory, but they cannot write to it. This means that any user can it cd to this directory and list the files, but they cannot create or delete files.

The file permissions are changed with the "chmod" command. You can use the letters u,g,o,a for users, group, others, or all and the letters w,r,x for read write and execute. A "+" adds permissions and a "-" subtracts permissions. For example if Frodo wanted to let all users have write permissions on his directory (a very bad idea), he would give the command:

chmod o+w bcCode

The corresponding line in the above listing would become:

drwxrwxrwx 2 frodo users ... bcCode

man Pages

One of the most useful commands in Linux is the man command. This is the Linux help system. The man pages are broken up into it NAME, SYNOPSIS, DESCRIPTION, OPTIONS, ENVIRONMENT, and SEE ALSO s. There may be some additional s, but these are the most common. The SYNOPSIS gives the syntax of the command with denoting optional parameters. Under the DESCRIPTION you will find a detail description of the action of the command in question. OPTIONS give a detailed look at each possible option for the command. In general Linux command options are preceded by a "-" or a "- -". The man page format becomes a very efficient help system once you become familiar with the layout. You will practice with these pages in the following exercises.
